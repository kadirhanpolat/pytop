Random Generators

Random and semi-random generators for foundational mathematical structures.

Provides: - random_set — frozenset with size control, element type, and ordering mode - LazyTopology — subbasis-backed topology with bitmask UID; no full enumeration - random_topology — random topology (exact for small carriers, lazy for large) - random_relation — random binary relation via density sampling - random_function — random function as a plain dict

exception pytop.random_generators.RandomGeneratorError

Bases: ValueError

Raised when a random generator receives invalid parameters.

pytop.random_generators.random_set(max_size: int | None = None, *, min_size: int = 0, size: int | None = None, element_type: str = 'int', pool: Iterable[Any] | None = None, random_order: bool = True, seed: int | None = None) → frozenset[Any]

Return a random frozenset of elements.

Parameters

max_size:

Upper bound on the number of elements. Ignored when size is given.

min_size:

Lower bound (default 0). Used only when max_size is given.

size:

Exact number of elements. Takes priority over min_size/max_size.

element_type:

"int" (default, draws from 1–99), "char" (a–z), or "str" (two-letter combinations aa–zz). Ignored when pool is given.

pool:

Custom iterable of elements to draw from.

random_order:

True (default): pool is shuffled — elements are chosen randomly. False: first k elements of the pool are taken in order.

seed:

Integer seed for reproducibility.

Examples

>>> random_set(max_size=5, seed=0)
frozenset({...})
>>> random_set(size=3, element_type='char', random_order=False)
frozenset({'a', 'b', 'c'})
class pytop.random_generators.LazyTopology(carrier: frozenset[Any], subbasis: frozenset[frozenset[Any]])

Bases: object

A topology represented by its subbasis — no full open-set enumeration.

Any collection of subsets generates a valid topology (the smallest one containing those sets), so LazyTopology is always mathematically correct by construction regardless of the subbasis chosen.

Each (carrier, subbasis) pair receives a unique integer UID built as a bitmask over the power set of the carrier:

Canonical order: carrier elements sorted by repr().
S_i = {carrier[j] : bit j of i = 1}   (the i-th subset in bitmask order)
UID = Σ  2^i  for all i where S_i ∈ subbasis

Parameters

carrier:

The underlying set X.

subbasis:

Generating collection of subsets of X.

pytop.random_generators.random_partial_order(carrier: Any, density: float = 0.5, seed: int | None = None) → set[tuple[Any, Any]]

Return a random partial order (reflexive, antisymmetric, transitive).

Algorithm:

  1. Pick a random permutation of carrier.

  2. For each pair (i, j) with i < j in the permutation, include the edge (perm[i], perm[j]) with probability density. This produces a DAG — no cycles possible.

  3. Apply transitive closure (antisymmetry preserved by the DAG structure).

  4. Add the diagonal to make the relation reflexive.

Parameters

carrier:

Elements of the underlying set.

density:

Probability that each forward DAG edge is included.

seed:

Random seed for reproducibility.

Return a random equivalence relation (reflexive, symmetric, transitive).

The carrier is randomly partitioned into k equivalence classes (k is chosen uniformly from 1 to |carrier|). Each element is independently assigned to a class. The resulting relation contains all pairs (a, b) where a and b belong to the same class.

Uses rejection sampling: uniform random functions are generated until a continuous one is found (verified via the basis preimage criterion). Works with both FiniteTopologicalSpace and LazyTopology.
